# 30_STEP011B_DOCS_INTEGRATION_DECISIONS

## Purpose

This document records what was taken from `md.zip`, what was not taken, and why.

## Directly integrated themes

- master operating manual → strengthened mode-first and first-response discipline
- docs pack spec → strengthened required-docs and anti-drift rules
- handoff standard → hardened current-project handoff structure
- mode selection guide → clarified primary / secondary mode rule
- new project starter → reinforced project-entry and narrow-step behavior

## Not promoted into main canon

- brand-system mode
- legal-docs mode
- GPT-builder notes
- GPT Builder copy-paste packs / operator prompts

## Why not everything was merged

This repo is a Telegram + engineering runtime track.
The main docs canon should strengthen execution, contracts, handoff, and continuity.
Anything that does not improve that directly stays out of the mainline docs spine.

## Main resulting changes

- `00_CURRENT_STATE.md` now explicitly records primary/secondary mode and STEP011B integration truth
- `README.md`, `00_BOOT.md`, `01_PROJECT_OPERATING_MANUAL.md`, `02_HANDOFF_STANDARD.md`, `03_DOCS_PACK_SPEC.md`, and `10_WORK_MODES.md` are tighter and less drift-prone
- `15_NEW_CHAT_HANDOFF.md` now exists as a ready-to-use continuation document
- `16_RELEASE_READINESS_CHECKLIST.md` now exists as a future-facing but concrete release gate
