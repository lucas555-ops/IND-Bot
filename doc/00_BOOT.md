# 00_BOOT

This project should be understood as:

**Telegram professional directory + LinkedIn identity bootstrap**

Not as:
- a LinkedIn clone
- a scraping tool
- an outreach automation tool
- a broad social network rewrite
- a hidden “growth automation” shell around unofficial LinkedIn access

## Primary / secondary mode

- Primary mode: **Telegram SaaS / Bot + Engineering Ops**
- Secondary mode: **Docs / Handoff / Hardening discipline**

## Donor DNA

- Collabka-first for router, profile, catalog, moderation thinking, and docs discipline
- CryptoCollabBot-secondary for compact UX, narrow-step hardening, and truthful status framing

## Operating rule

- One narrow STEP at a time
- Truth before momentum
- Reversible moves before broad rewrites
- Docs and contracts should lead runtime, not trail it
- Handoff should be explicit, not reconstructed from memory

## Docs canon

Read in this order:
1. `docs/README.md`
2. `docs/00_CURRENT_STATE.md`
3. `docs/01_PROJECT_OPERATING_MANUAL.md`
4. `docs/04_EXECUTION_CONTRACT.md`
5. `docs/06_TRUTH_BOUNDARY_AND_SPIKE_RULES.md`
6. `docs/08_TELEGRAM_UI_ROUTER_CONTRACT.md`
7. `docs/09_SELECTION_SURFACES_CONTRACT.md`
8. `docs/15_NEW_CHAT_HANDOFF.md` when continuing in a new chat

## Hard boundaries

- Official LinkedIn OIDC only for auth/identity bootstrap
- No scraping DNA in core runtime
- No fake production claims
- No reply-chain Telegram UX drift
- No money/admin/policy surface without explicit contract
- No docs drift between current state, feature baselines, and work history
